## Overview

In this room, I learned the fundamentals of cloud computing and how modern organizations use cloud platforms to run applications more efficiently.

Cloud computing did not appear overnight. It evolved from traditional physical servers as businesses looked for ways to reduce costs, improve scalability, and make their applications easier to manage.

Today, cloud computing powers many of the world's most popular services, including Netflix, Spotify, Instagram, and countless business applications.

This room introduced cloud deployment models, cloud service models, major cloud providers, and the key benefits that make cloud computing one of the most important technologies in modern IT and cybersecurity.

---

## How Servers Evolved to the Cloud

Before cloud computing, organizations relied entirely on physical servers.

### Traditional Physical Servers

Characteristics:

- One application per server
- High hardware costs
- Limited scalability
- Difficult maintenance
- Resource wastage

As businesses grew, this model became expensive and inefficient.

---

### Virtualisation

The introduction of virtualization allowed multiple virtual machines to run on a single physical server.

Benefits included:

- Better hardware utilization
- Lower costs
- Easier management
- Faster deployment

---

### Cloud Computing

Cloud computing took virtualization a step further by allowing organizations to access computing resources over the internet whenever needed.

Instead of buying servers, companies can now rent resources on demand.

---

## What is Cloud Computing?

Cloud computing is the delivery of computing services over the internet.

These services include:

- Servers
- Storage
- Databases
- Networking
- Software
- Analytics
- Security Services

Users can access these resources without owning the physical infrastructure.

---

# Benefits of Cloud Computing

Cloud computing offers several advantages over traditional infrastructure.

---

## 1. Scalability

Cloud resources can easily scale up or down based on demand.

Example:

An online store can increase resources during Black Friday and reduce them afterward.

Benefits:

- Flexible growth
- Improved performance
- Reduced waste

---

## 2. On-Demand Self-Service

Resources can be created or removed instantly.

Examples:

- Virtual servers
- Databases
- Storage
- Applications

No hardware installation is required.

---

## 3. Pay Only for What You Use

Cloud providers charge based on actual usage.

Benefits:

- Lower upfront costs
- Better budgeting
- Reduced waste

Organizations avoid purchasing expensive hardware that may sit unused.

---

## 4. Security

Cloud providers implement strong security controls.

Examples include:

- Encryption
- Identity Management
- Firewalls
- Monitoring
- Physical Security

Security remains a shared responsibility between the provider and the customer.

---

## 5. High Availability

Cloud platforms are designed to remain operational even when failures occur.

Benefits:

- Reduced downtime
- Improved reliability
- Better business continuity

---

## 6. Global Access

Applications can be accessed from anywhere in the world.

Benefits:

- Faster global delivery
- Improved user experience
- Worldwide availability

# Types of Cloud Deployment Models

Organizations can deploy cloud environments in different ways depending on their needs.

---

## Public Cloud

A Public Cloud is shared infrastructure provided over the internet.

Examples:

- AWS
- Microsoft Azure
- Google Cloud Platform

### Characteristics

- Cost-effective
- Highly scalable
- Easy to deploy
- Managed by the provider

### Common Use Cases

- Websites
- Startups
- Mobile Applications
- SaaS Platforms

Public cloud services are suitable for most modern applications.

---

## Private Cloud

A Private Cloud is dedicated to a single organization.

### Characteristics

- Greater control
- Increased customization
- Enhanced compliance
- Improved data governance

### Common Use Cases

- Government Agencies
- Healthcare Organizations
- Banks
- Financial Institutions

Private clouds are often used when sensitive data requires strict controls.

---

## Hybrid Cloud

A Hybrid Cloud combines Public and Private Cloud environments.

### Characteristics

- Greater flexibility
- Sensitive data remains private
- Public cloud scalability available

### Common Use Cases

Organizations that:

- Store sensitive information privately
- Need additional resources during peak demand

Example:

An e-commerce company may keep customer data private while using public cloud resources during major sales events.

---

# Cloud Service Models

Cloud providers offer services at different levels depending on how much responsibility the customer wants to manage.

---

## Infrastructure as a Service (IaaS)

IaaS provides basic computing resources.

Examples:

- Virtual Servers
- Storage
- Networking

### Customer Manages

- Operating Systems
- Applications
- Security Configuration

### Provider Manages

- Physical Hardware
- Data Centers
- Infrastructure

Example:

AWS EC2

---

## Platform as a Service (PaaS)

PaaS provides a complete platform for developing and deploying applications.

### Customer Manages

- Application Code
- Data

### Provider Manages

- Servers
- Operating Systems
- Infrastructure

Benefits:

- Faster development
- Reduced maintenance
- Simplified deployment

---

## Software as a Service (SaaS)

SaaS delivers complete software applications over the internet.

### Provider Manages Everything

Users simply access the application.

Examples:

- Gmail
- Zoom
- Microsoft 365
- Dropbox

Benefits:

- No installation required
- Easy access
- Automatic updates
